Om prosjektet

Holidaze er en fiktiv side jeg utviklet som eksamensprosjekt på studiet ved Noroff. Målet var å lage en moderne, brukervennlig og responsiv plattform for booking av overnattingssteder, både for vanlige brukere og for de som eier og vil leie ut et sted.

Formål og målgruppe: Plattformen er laget for både gjester som ønsker å booke overnatting, og for eiere som vil administrere sine utleiesteder.

Hovedfunksjoner

  • Registrering og innlogging for ulike brukerroller
  • Søk og filtrering av overnattingssteder
  • Booking og oversikt over reservasjoner
  • Opprettelse og redigering av egne overnattingssteder
  • Responsivt og brukervennlig design

Teknologi og verktøy brukt

  • React
  • Tailwind CSS
  • React Router
  • Noroff API
  • Figma
  • Netlify

Resultat

Dette prosjektet har gitt meg god erfaring med å bygge en komplett side fra bunnen av. Jeg har fått jobbet med api, tilpasset løsningen til ulike brukerroller, og lært mye om hvordan man lager en app som både ser bra ut og fungerer godt i praksis.

Refleksjoner og forbedringer

Hva lærte jeg? Dette prosjektet ga meg en dypere forståelse for hvordan man bygger en komplett side fra bunnen av. Jeg fikk testet ut React i praksis, jobbet mye med api og lærte hvor viktig det er å tenke på brukeropplevelse og tilgjengelighet gjennom hele prosessen. Det var også lærerikt å jobbe med autentisering og ulike brukerroller.

Hva ville jeg gjort annerledes? Jeg ville brukt mer tid på planlegging og strukturering av komponenter i starten, for å gjøre det lettere å utvide og vedlikeholde koden senere. Jeg ser også at det er lurt å involvere brukertesting tidligere i prosessen for å fange opp småfeil og forbedringsmuligheter.

Hvordan kan prosjektet forbedres videre? Jeg kunne tenkt meg å legge til flere filtreringsmuligheter, bedre feilhåndtering og kanskje en kalenderfunksjon for booking. Det hadde også vært spennende å jobbe videre med design og animasjoner for å gjøre opplevelsen enda mer engasjerende.
